# Model Benchmarking & Selection Architecture

## Overview
This document summarizes the comparative evaluation of multiple classification architectures against the e-commerce session dataset. The primary objective was to deploy a robust estimator that maximizes the identification of high-intent conversion sessions while maintaining acceptable precision.

## Evaluation Framework
Models were assessed using a stratified holdout validation strategy to preserve class distributions. Given the imbalanced nature of conversion events, baseline Accuracy is a fundamentally flawed metric. Model selection was therefore strictly gated on **Weighted F1 Score** and **ROC AUC**, providing a balanced assessment of precision-recall tradeoffs across both majority and minority classes.

## Benchmarked Architectures

### 1. Baseline: Logistic Regression
- **Configuration**: L2 Regularization (Ridge), 1000 max iterations.
- **Hypothesis**: Linear decision boundaries may suffice if the feature space, post-SMOTE synthesis, exhibits linear separability.
- **Observations**: Logistic Regression establishes a solid computational baseline but struggles to capture complex, non-linear interactions between variables like `PageValues` and `ExitRates`.

### 2. Challenger: Random Forest
- **Configuration**: 100 Estimators, Gini Impurity criterion.
- **Hypothesis**: An ensemble of deep decision trees inherently models non-linear relationships and interactions without requiring explicit polynomial feature engineering.
- **Observations**: Substantial performance increase over the linear baseline. The bagging approach significantly reduces variance, though the estimator suffers from higher computational inference latency compared to boosting algorithms.

### 3. Primary: XGBoost (Extreme Gradient Boosting)
- **Configuration**: Logloss evaluation metric, native API integration.
- **Hypothesis**: Sequential boosting iteratively minimizes residual loss, allowing the model to focus rigorously on difficult-to-classify edge cases (e.g., high-intent sessions that abandon late).
- **Observations**: XGBoost consistently dominates the performance frontier. It rapidly isolates dominant interaction effects and efficiently processes the sparse matrices generated by the `OneHotEncoder`.

## Selection Justification
Based on the quantitative metrics generated during the benchmarking phase, the **XGBoost Classifier** was selected for production deployment within the Inference Engine. 
The tree-based architecture is highly resilient to collinearity and requires no further normalization of the scaled inputs. The final pipeline effectively balances computational latency (critical for real-time session inference) with state-of-the-art predictive capability, ensuring that downstream marketing automation systems receive highly calibrated intent probabilities.
